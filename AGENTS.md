# Paddock Atlas

A **Jurassic World Evolution 3** park planner. You pick species for an enclosure; it live-scores
every other species and derives a build order (plants, terrain, fence, feeders, population).

Next.js 16 (App Router) · React 19 · TypeScript · Tailwind v4 · JSON data · client-only with
`localStorage`. No backend, no tests yet.

## Commands

```bash
npm run dev              # dev server (Turbopack). Config: .claude/launch.json
npm run build            # production build — run before declaring work done
npm run lint             # eslint (must be clean; it catches setState-in-effect etc.)
npx tsc --noEmit         # typecheck
node scripts/ingest.mjs  # regenerate data/ from scraper/jwe3_dinos.json
```

Routes: `/` (the app) · `/states` (live §9 states reference).

## Hard rules

1. **Never hand-edit `data/species.json`, `rulesets.json`, `formations.json`, `manifest.json`.**
   They are generated by `scripts/ingest.mjs`. Edit the ingest script or the raw scrape, then
   re-run it. Hand edits are silently destroyed on the next run.
2. **Never invent game data.** Species stats, relationships, needs, appeal and periods all come
   from the scrape. If a fact isn't in `scraper/jwe3_dinos.json`, the app must not claim it.
   `data/plants.json` and `data/periods.json` are the only hand-authored data, and both say so.
3. **Never store derived state.** Candidates, requirements and `isParkCustom` are computed from
   the roster + ruleset on every render. Adding a cached/duplicated field is how this app gets
   wrong. See `lib/AGENTS.md`.
4. **Never use raw colors or Tailwind's default palette** (`bg-red-500`, `#111`, inline
   `oklch(...)`). Only role tokens from `app/globals.css`. See `components/AGENTS.md`.
5. **Keep the theme bootstrap as `next/script` with `strategy="beforeInteractive"`**
   (`app/layout.tsx`). A plain `<script>` in the component tree triggers a React dev warning;
   loading it later than `beforeInteractive` reintroduces a theme flash on load.

## What the dataset does NOT contain

The source is paleo.gg DinoDB (Source A) only. It has **no canon film eras, no in-game model
variants, no precise mya ranges, and no flora table**. Canon-era rulesets and the variant picker
were therefore deliberately **removed** — the reality layer runs on geologic **Period** and
dig-site **Formation** instead.

Do not re-add canon/variant features, and do not fabricate that data to make a design mock match.
If someone supplies the FatherMC accuracy guide (Source B), that's what unlocks it.

## Verify before you finish

`tsc` + `lint` + `build` clean, **and drive the actual UI** — this app's value is live
re-scoring, which type checks can't prove. Start the dev server, change a roster, and confirm
candidates and Build Requirements both move.

`/states` renders every edge case (empty park, blocked candidate, conflict, population violation,
marine, aviary, juvenile, data drift, lab hybrid) from real data — it's the fastest way to check
you didn't break a state you weren't thinking about.

Known tooling quirk: screenshots of the tall `/states` page can come back black after scrolling.
That's a browser-pane compositor artifact, not a bug — verify that page by extracting DOM text or
using a taller viewport at scroll 0.

## Design source of truth

Built from a Claude Design handoff (`~/Downloads/Paddock Atlas Design Specification-handoff.zip`).
Chosen production skin is **Classic Jurassic Park dark**; the light skin is a retained reference.

Where the build deliberately departs from the mock — and why — is recorded in
[README.md](README.md) under "Notes on design fidelity". Read that before "fixing" something to
match a screenshot. Notably: the mock's T. rex + Triceratops + Ankylosaurus roster is impossible
on real data, because T. rex genuinely dislikes both.
